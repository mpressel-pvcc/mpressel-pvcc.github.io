Chapter 5: Programming Languages
Machine Language An encoding system that sequences instructions using numeric digits. Very tedious.
Debugging Locating and correcting errors in a program.
Assembler Converts expressions into machine language.
Assembly Language A mnemomic system for representing computer programs.
Translator Converts high-level programming language into machine language
Compiler Converts multiple sets of machine instructions into shorter sequences to replicate the behavior requested by the high-level primitive. C++ is a compiled language.
Interpreter Programs similar to translators. Execute instructions as they are translated instead of saving it all for the end. Python is an interpreted language.
Cross-Platform Software Software designed to be used on different types of machine hardware as well as different operating systems. Executable throughout a network.
Natural Language Languages that have evolved over years without formal grammar rules. Human languages are natural.
Formal Language Language designed with specific rules from the start. Programming languages are Formal.
Machine Independence Being able to run a program on different types of machines as long as they had the appropriate compiler.
High-Level Programming Language Language that allows a program to be written in terms of the problem instead of in terms of the machine.
Object-Oriented Programming (OOP) A paradigm where a software system is viewed as a group of units able to provide functionality themselves, but also work in tandem with other units (objects)
Scripting Language Usually used for administrative tasks rather than complex programs.
Data Types:
  • Integer: Numeric data consisting of whole numbers.

  • Float: Numeric data that may contain both whole and non-whole numbers (floating point notation)

  • String: Data that uses collections of characters.

Primitive Data Type A basic data type that is built into a language so to speak, such as int, char, float, bool, etc.
Data Structure How information is arranged. Such as a string, array, etc.
Python Collection Data Structures
  • List: uses [] and is an ordered (can be indexed) and changeable (add, remove, clear, etc) collection type.

  • Tuple: Uses () and is an ordered, but unchangable data collection type. Think a list of unchanging rates.

  • Set: uses {} and is an unordered, unchangable (as in data in it cannot be changed. Data can be added or removed), and unindexed data collection.

  • Dictionary: uses {} and is an ordere and changeable data collection that uses key:value pairs and does not allow duplicates.
Function A chunk of code that does a thing that can be used by other parts of a program.
Local Variable A variable that only has scope within the function it is defined in.
Global Variable A variable that has scope within a whole program. Often many functions can access and change global variables.
Source Program The code that is written in a high-level programming language.
Object Program What source code is translated into by a compiler of assembler.
