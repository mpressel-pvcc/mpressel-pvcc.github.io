Operating System Terminology
TERM DEFINITION
Operating System The software that controls the overall operation of the computer; including files management, the user interface, and the program execution environment.
Job Queue A storage organization used by the operating system to handle how jobs get executed. Most job queues will take job priority into consideration rather than adhere strictly to FIFO.
FIFO First-in, first-out. Where objects are removed from the queue in the order in which they arrived.
Interactive Processing A feature in an operating system that allows a user and the program being executed to have a dialogue through remote terminals.
Time-Sharing The concept that an operating system can provide service to multiple users at the same time.
Multiprogramming An implementation of time sharing where time is divided into intervals and each job execution is restricted to one of the time intervals. At the end of each interval, the operating systems moves to the next job, creating the illusion of several jobs being executed at once.
Application Software The programs for performing tasks particular to a machine's utilization. i.e. spreadsheets, accounting systems, games, development software, etc.
System Software The programs for performing tasks that are common to computer systems in general. The infrastructure that is necessary to use application software.
Utility Software A type of system software that performs activities that are fundamental to computer installations, but not included in the operating system. Examples would be programs used to format a disk, copy files from a magnetic disk or other storage device, compress or decompress data, or other cusomtizations.
Graphical User Interface (GUI) The part of the operating system that allows a computer's users to manipulate objects, such as files and programs, that are represented pictorially on the display. Users can interact with these objects in multiple ways. By using the mouse, keyboard, touch screen, or other input device.
Shell Older uder interfaces that communicated with users through textual messages using a keyboard and monitor screen.
Kernel The internal part of the operating system. It contains the software components that perform basic functions required by the computer installation.
Device Driver A software component of the kernel used to communicate with the controllers to carry out operations on the peripheral devices attached to the machine. Each driver is designed for a specfic device (Printer, disk drive, monitor, etc.)
File Manager A software component of the kernel used to coordinate the use of a machine's mass storage. Maintains records of all files stored, including where it is stored, which users can access the files, and which areas of mass storage are available for new files.
Memory Management A coponent of the kernel that executes the task of coordinating the machine's use of main memory. The needs are extensive when dealing with multiuser or multitasking environments.
Virtual Memory An illusion of having extra memory created by dividing data into uniformly sized units (called pages,) and then rotating those pages back and forth between main memory and mass storage (a process called paging.)
Boot Strapping Often called booting, this process occurs everytime the computer is turned on. It transfers the operating system from mass storage (where it is permanently stored) into main memory (which is basically empty when the machine is first turned on.)
ROM Nonvolatile memory cells in main memory where the CPU expects to find its initial program.
Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource allocated to another.
Spooling The technique of holding data for output at a more convenient time.
Super User (or Administrator) A person with highly priveleged access to the operating system by identifying themself as the administrator (typically via username and password) and can alter settings within the operating system and perform other tasks that are denied to normal users.
Multi-Factor Authentication When an account requires more than one piece of evidence to gain access. For example, a password and security questions, or a password and a code emailed to you.
