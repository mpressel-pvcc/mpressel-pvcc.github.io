All About GitHub

Q&A

  • Q. What is the version control system?
  • A. A version control system (or VCS) tracks the history of changes to a project as individuals or teams make edits or updates. As changes are made, earlier versions can be recovered at any time

  • Q. What is Git?
  • A. Git is an open source, well maintained, revision control system that is used for source code management.

  • Q. What is GitHub?
  • A. GitHub is a code hosting platform for version control and collaboration. It is cloud based and allows for people all over the world to collaborate and work on projects.

  • Q. What is the difference between Git and GitHub?
  • A. Git is the version control system used to manage source code history. GitHub is a place where source code and projects can be uploaded to a repository and collaborated on.

  • Q. Who started GitHub and how was it started?
  • A. GitHub was started by Chris Wanstrath, P. J. Hyett, Tom Preston-Werner, and Scott Chacon in 2008 in San Fransisco. It was developed on Ruby on Rails for the creators to work on their own projects; specifically to collaborate with each other. It has been paramount in developing open source software.

  • Q. What company owns it now?
  • A. Microsoft acquired GitHub in 2018 for $7.5 billion in one of the largest deals of that year.

  • Q. How much does a GitHub account cost?
  • A. There are three tiers of membership. A free option for individuals and organizations, and mid-tier level for advanced collaboration ($44/year), and an enterprise level ($231/year) for security, compliance, and flexible deployment.

  • Q. What is the Octocat?
  • The Octocat is the mascot of GitHub. Created from a cartoon by David O'Reilly.

Git/GitHub Terms

  1. Repository - Contains all of you r project files and each file's revision history. You can manage and discuss within the repository.
  2. Commit - Records changes to your repository on a local level.
  3. Fork - A new repository that shares code and visibility settings with the original "upstream" repository.
  4. Push - Updates your git commit changes and sends it to a remote repository.
  5. Pull Requests- Pull requests let you inform onther people, usually within a shared or forked repository, that you have made changes and pushed them. Collaborators can then follow up and review your changes.
  6. Workflows- Are a configurable automated process that allows you to run one or more jobs. The actions can be defined and executed based on predetermined triggers defined in your repository.
  7. Issues- Issues let you track work on your development projects. They can be used to cross reference work and can be linked to a pull request.
  8. Raw Button- The raw button allows a user to view their project without any styling.
  9. Blame Button- The blame button lets you see the entire revision history line-by-line. Each time you click you will see the previous revision and who comitted the change.
